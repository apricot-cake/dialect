# dialect

各SNS・サイトの検索を「演算子つきディープリンク」に翻訳して開くツール。設計・背景は [README](README.md) と `docs/`、演算子の定期確認は `.claude/skills/operator-check` を参照。

## 対応サイトの追加・調査は表層で止めない（網羅してから「済み」とする）

新しい対応サイトを追加する、または既存サイトの対応演算子を調べ直すときは、**そのサイトの検索フォーム・詳細検索・フィルタのGUIパネルを実際に開き、各オプションを操作して生成URLを採取してから**「採取済み・網羅済み」と記録・報告すること。結果ページのリンクや、既定で表示される並び順コントロールだけで済ませない。表層で止めたものは「済み」ではなく「未」。

判定基準は「自分が見た範囲」ではなく「そのサイトのUIが実際に提供している全オプション」。迷えば「未」に倒す。「詳細検索やフィルタを開かないと出てこなかったから」を網羅の免罪符にしない。

最低限の採取手順:

- 検索ボックスに実入力して送信し、生成される検索URLを見る
- 並び順・絞り込みの各コントロールを操作し、それぞれの生成URL（パラメータ）を採取する
- 「詳細検索」「フィルタ」等のパネルを開き、全項目（日付・投稿タイプ・除外・送信者・タグ 等）を操作して生成URLを採取する
- 要ログインのサイトはログイン済み状態でも確認する

**実例（2026-07-07）**: Tumblr を追加したとき、この手順を飛ばして結果ページのリンクと並び順だけ採取し「実機確認済み」と報告した。後で詳細検索GUIを開いたら `from:`（送信者）／`-`（除外）／`since:`・`before:`（期間）／複数タグ／完全一致／`postTypes=`（投稿タイプ）を丸ごと取りこぼしていたことが判明。ニコニコ静画は詳細オプションが無いことを確認済み＝網羅済み。

## 演算子調査の出所を記録する（doc / URL叩き / GUI操作 の区別）

`docs/operator-research.md`・`docs/operator-checklist.md` に演算子の所見を書くときは、**その情報をどうやって得たか**を明記する。「実測」という言葉だけでは「公式ドキュメントの記載」「URLパラメータを直接組んで件数比較しただけ」「実際にそのサイトのGUI（検索ボックス・並び順・詳細検索/フィルタ）を操作して確認した」の3つが区別できず、上の「表層で止めない」ルールの徹底度を後から検証できない。

記録には最低限、次のいずれかのタグと日付を添える:

- **doc**: 公式ドキュメント・公式ヘルプに記載がある（URLも残す）
- **URL叩き**: URLパラメータを直接組んで開き、件数比較や表示結果で動作を確認した（GUI操作はしていない）
- **GUI操作**: 実際にそのサイトの検索フォーム・並び順・詳細検索/フィルタパネルを操作し、生成されたURLを採取した（このルールの「表層で止めない」を満たす最も強い根拠）

「GUI操作」タグが無い所見は、まだ表層調査の可能性がある＝将来の監査対象として扱う。

## フィールドの採否選定はしない（網羅実装＋機構で見せ方を解決）

実在すると確認できた検索オプション（URLパラメータ化できるもの）は、**「有益かどうか」を人間が選んでから実装する二段構えを取らない**。マニアックさ・使用頻度・UIノイズを理由に実装を見送らない。全部実装したうえで、ユーザーへの見せ方（探索しやすさ・UIの煩雑さ回避）はピッカーの機構（セマンティック検索・カテゴリフィルタ・家族タグによる姉妹提案/まとめて追加）側で解決する。

**背景**: 当初は「網羅収集→有益なものだけ選定→採用」の二段方針だったが、2026-07-05〜06の設計改訂でこの二段目（人による選定）を機構に置き換えることを確定した（verifiedOnly の削除撤回、旧不採用だった制作ツール/YouTube特徴/多言語の復活が先例）。**2026-07-07、既存サイトのGUI再監査で見つかった新規発見（Xメンション検索・Reddit type=・pixiv s_mode=tag_tc・YouTube特徴群）に対して、この確定済み方針を忘れて再び「採否をユーザーに確認してから」という二段構えを提案してしまった**。この種の後戻りを防ぐため、ここに明文化する。

**適用**: 新サイト追加時・既存サイトの監査時とも、実在確認できたURLパラメータは見送り候補を作らず実装する。実装コスト（命名・説明文・日英i18n・実機確認などの編集コスト）自体は残るため作業の分割・優先順位付けはしてよいが、「これは載せない」という取捨選択はしない。
